### 审核工作台

  构建统一的复核工作台，聚合入库、出库、调拨等全业务链路的审核流程。
  通过 Handler 模式实现业务解耦，新增业务类型只需实现对应 Handler 即可接入工作台。
  抽象出状态流转、库存操作等 5 大能力接口，建立业务开发规范。
  解决了多菜单重复审核问题，实现工作台与业务表的实时双向同步。

  技术实现：
  1. 采用策略模式+Handler 包装器模式，通过业务类型路由到对应的审核处理器（Handler），屏蔽各业务实现差异
  2. 利用 Spring 依赖注入自动收集 Handler 实现类，通过 @PostConstruct 构建路由 Map
  3. 实现工作台与业务表的双向同步，确保数据一致性

  架构价值：建立了可扩展的能力接口体系，为后续业务标准化开发提供规范指导，新增业务只需实现对应接口即可快速接入


  数据指标
  - 聚合了 20 种业务类型
  - 提升了 200% 多少审核效率
